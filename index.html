STGI Hackathon

Build Your Own Observability Platform

Background

In modern cloud environments, monitoring and observability are crucial for keeping applications running smoothly. In this hackathon, your challenge is to create a basic cloud monitoring tool. It will collect, process, and display logs and metrics from a Kubernetes cluster (control plane as well as pods).

Challenge

You will set up a Kubernetes cluster (using Minikube) with two applications, a PostgreSQL and a MySQL database, running on your local machine. Your task is to build a monitoring solution that collects logs from the applications running in the Minikube cluster, authenticates them, stores them in a time series database, and visualizes them in a custom dashboard.

Requirements

  • Initial Setup: Create a Minikube cluster hosting two apps and databases.
  • Log Collection: Use Fluent Bit, FluentD, or Logstash to gather logs from application pods.
  • Backend Service: Develop a backend service to receive logs and implement authentication.
  • Time Series Database: Store and organize logs efficiently.
  • Visualization Dashboard: Build a custom web-based dashboard for log display and analysis.
  • Database Metrics Monitoring: Collect and visualize PostgreSQL database metrics.
  • AWS Integration: Integrate AWS resources and visualize combined metrics.
  • Efficient Search (Bonus): Add keyword-based search and filtering capabilities.

Suggested Tools

For web development and visualization, you can use any stack like React, Django, Go, Node, etc.

Initial Setup

  • Install Docker: Get Docker
  • Install and Start Minikube: Minikube Installation Guide
  • Clone Sample Applications:
    • app-1: PyEditorial
    • app-2: Docker Node CRUD MySQL
  • Create Kubernetes Deployments and Services for both applications and databases.
  • Perform a sanity check to verify that all apps and databases are running correctly.

Evaluation Criteria

Your project will be evaluated based on the following:

  • Functionality: Effective log collection, processing, and visualization.
  • Usability: User-friendly dashboard interface.
  • Completeness: Well-structured insights provided by the tool.
  • Bonus: Efficient search and filtering capabilities.
